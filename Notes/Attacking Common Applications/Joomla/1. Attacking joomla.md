## Leveraging Known Vulnerabilities

We may get a lot of listings as CVEs for Joomla, but all of them wont give you RCE.
Searching a site such asô `exploit-db`ô shows over 1,400 entries for Joomla, with the vast majority being for Joomla extensions.

Researching a bit, we find that this version of Joomla is likely vulnerable toô [CVE-2019-10945](https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2019-10945)ô which is a directory traversal and authenticated file deletion vulnerability. 

We can useô [this](https://www.exploit-db.com/exploits/46710)ô exploit script to leverage the vulnerability and list the contents of the webroot and other directories. 

**The python3 version of this same script can be foundô [here](https://github.com/dpgg101/CVE-2019-10945).**

This could lead to access to sensitive files such as a configuration file or script holding credentials if we can then access it via the application URL. An attacker could also cause damage by deleting necessary files if the webserver user has the proper permissions.

We can run the script by specifying theô `--url`,ô `--username`,ô `--password`, andô `--dir`ô flags.

```bash
# https://www.exploit-db.com/exploits/46710
python2.7 joomla_dir_trav.py --url "http://dev.inlanefreight.local/administrator/" --username admin --password admin --dir /

# https://github.com/dpgg101/CVE-2019-10945
python3 CVE-2019-10945.py --url "http://dev.inlanefreight.local/administrator/" --username admin --password admin --dir /
```
As pentesters, this would only be useful to us if the admin login portal is not accessible from the outside since, armed with admin creds, we can gain remote code execution, as we saw above.
