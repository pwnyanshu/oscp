
Burp's web fuzzer is called `Burp Intruder`, and can be used to fuzz pages, directories, sub-domains, parameters, parameters values, and many other things.

This may also act as an alternative for many of the CLI-based fuzzers we use, like `ffuf`, `dirbuster`, `gobuster`, `wfuzz`, among others.

---

**Burp Intruder – Quick Notes**

**Purpose**

- Burp Intruder is Burp’s built-in web fuzzer.
    
- Used for fuzzing directories, parameters, brute-forcing, and enumeration.
    
- Free version is slow (1 req/sec). Pro is fast.
    

---

**Workflow**

1. Capture request in Proxy → right-click → _Send to Intruder_ (`CTRL+I`).
    
2. Go to **Intruder** tab → configure:
    
    - **Target** – auto-filled from request.
        
    - **Positions** – mark fuzzing point using `§` (e.g., `/§DIRECTORY§/`).
        
    - **Payloads** – load wordlist and configure processing.
        
    - **Options** – configure Grep-Match, retries, encoding.
        

---

**Positions**

- Select the keyword (e.g., DIRECTORY) → _Add §_.
    
- Leave two blank lines at the end of the request (required by Burp).
    

---

**Payloads**

- **Payload Type:** Simple List (most common).
    
- **Load Wordlist:** `/opt/useful/seclists/Discovery/Web-Content/common.txt`
    
- **Runtime file** is better for huge wordlists (saves RAM).
    

---

**Payload Processing**  
Examples:

- Skip lines starting with `.`:
    
    - Add rule → “Skip if matches regex”
        
    - Regex: `^\..*$`
        
- Append text (e.g., `.html` suffix):
    
    - Add rule → “Add prefix/suffix”
        

---

**Payload Encoding**

- URL-encoding is enabled by default.
    
- Only disable encoding when testing applications expecting raw characters.
    

---

**Settings**

- Set **Network retries = 0** and **Pause before retry = 0** for speed.
    
- **Grep – Match:** highlight specific responses (e.g., `200 OK`).
    
    - Enable → Clear list → Add “200 OK”
        
    - Turn off _Exclude HTTP Headers_ so you match status lines.
        

---

**Finding Valid Directories**

- Run attack → sort by:
    
    - **Status** (look for 200 responses)
        
    - **Length** (different length = potential hit)
        

Example: Fuzzing `/admin/§1§.html` may reveal valid pages.

---

**Key Shortcuts**

- Send to Intruder: `CTRL+I`
    
- Go to Intruder: `CTRL+SHIFT+I`
    
- URL encode: `CTRL+U`
    
- URL decode: `CTRL+SHIFT+U`
    

---

**Typical Use Cases**

- Directory fuzzing
    
- Parameter fuzzing
    
- Login brute-force (slow in free version)
    
- File discovery (e.g., `.html`, `.php`)
    
- API enumeration
