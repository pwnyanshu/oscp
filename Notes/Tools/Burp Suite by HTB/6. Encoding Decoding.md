
**Encoding / Decoding – Short Notes**

**1. Why Encoding Matters**

- When modifying HTTP requests, special characters can break the request.
    
- Encoding ensures the server interprets data correctly.
    
- Decoding helps us read encoded values from cookies, parameters, or responses.
    

**2. URL Encoding**  
Used to safely send special characters in URLs.

Common characters needing encoding:

- Space → `%20`
    
- `&` → `%26`
    
- `#` → `%23`
    

**Burp:**

- Right-click → Convert Selection → URL → URL-encode key characters
    
- Shortcut: `CTRL+U`
    
- `CTRL+SHIFT+U` → decode
    

    

**3. Common Encodings**

- **URL encoding** → `%xx`
    
- **HTML encoding** → `&lt;`, `&gt;`
    
- **Unicode** → `\uXXXX`
    
- **Base64** → Used heavily in cookies, tokens, API data
    
- **ASCII hex** → `41` → `A`
    

**4. Decoding Workflow**

- Identify encoded text
    
- Try Base64, URL-decode, HTML-decode, Unicode
    
- Tools: Burp Decoder, Burp Inspector, ZAP Encoder/Decoder
    

**Example Base64 Cookie:**

```
eyJ1c2VybmFtZSI6Imd1ZXN0IiwgImlzX2FkbWluIjpmYWxzZX0=
```

Decodes to:

```
{"username":"guest","is_admin":false}
```

**5. Re-Encoding**  
After modifying decoded values:

- Re-encode **in the same format** (Base64 etc.)
    
- Replace original cookie/parameter in the request
    

**Use-case:**  
Change `guest → admin`, `false → true`, encode again, send request.

**6. Tools**  
**Burp:**

- Decoder tab
    
- Inspector automatically decodes
    
- One-click re-encoding
